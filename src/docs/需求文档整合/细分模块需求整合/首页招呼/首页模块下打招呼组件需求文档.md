# 首页打招呼组件（简化版 PRD）

## 文档信息
- 文档版本：v1.0-简化
- 对接组件：`src\modules\Home\components\Greeting\index.tsx`
- 适用平台：App（安卓和 iOS）

## 产品原则
- 自然、克制、亲和，不打扰。
- 个性化但注意隐私与合规；不使用夸张和营销化表达。
- 中文单句，≤30 字；不使用 emoji；标点不超过 2 个。

## 范围与边界
- In Scope
  - App 打开首页或回到前台时，生成并展示一条问候语。
  - 信号源：用户昵称、校区名称、校区天气摘要、当前时间段标签。
  - 生成方式：将信号源合并后投喂大模型，由大模型输出 ≤30 字中文招呼语。
  - 支持手动刷新一次（小按钮），触发重新拉取天气与重新生成。
  - 降级：若大模型或天气接口失败，使用模板拼接方式生成可读招呼语。
- Out of Scope（本期不做）
  - 语音播报、图片/动效、个性化表情/emoji。
  - 通知推送与定时提醒。
  - 多轮对话、长期记忆与推荐。
  - 跨设备/服务器侧数据留存与个性化分析。

## 核心概念与数据模型
- Greeting（招呼语）
  - 字段
    - `content`: string（最终展示文案，≤30 字）
    - `createdAt`: ISO string（生成时间）
    - `signals`: object（用于生成的核心信号，仅本地使用）
      - `name`: string（优先使用昵称；缺省用“同学”）
      - `campus`: string（如“雅安校区”“成都校区”等）
      - `weather`: { `summary`: string; `temp`?: string }（如“小雨”“晴朗”“多云，18℃”）
      - `timeSlot`: `'凌晨'|'清晨'|'上午'|'中午'|'午后'|'傍晚'|'夜间'`
  - 约束
    - content 必须为简体中文，长度 ≤30 字；不得包含 emoji；语气自然友好。
    - 不得出现宗教、政治、广告等敏感信息；不泄露隐私数据。
    - 大模型失败则使用降级模板；确保始终有可展示内容。

## 数据来源与规则
- 用户名（name）
  - 来源：用户资料中的昵称；若无则退化为“同学”。
  - 隐私：仅将昵称传给大模型，不上传真实学号/手机号等敏感信息至日志。
- 校区（campus）
  - 来源：用户设置页选定的校区；如未设置，使用上次成功定位的城市/区名；均不可用则默认“雅安校区”。
  - 变更：用户切换校区后立即刷新招呼语。
- 天气（weather）
  - 接口：对接统一天气服务（由工程侧实现，可基于城市/区名称查询）。
  - 期望输出：`summary`（天气现象中文，如“晴”“小雨”“多云”）、可选 `temp`（如“12℃”）。
  - 频率与缓存：首页进入时请求；缓存 30 分钟；超过缓存或手动刷新才再次请求。
  - 失败处理：若获取失败，`weather.summary` 置为 `''`，降级模板中省略天气短语。
- 时间段（timeSlot）
  - 规则（本地计算，24h）：
    - 00:00–05:00：`凌晨`
    - 05:00–08:00：`清晨`
    - 08:00–11:00：`上午`
    - 11:00–13:00：`中午`
    - 13:00–17:00：`午后`
    - 17:00–19:00：`傍晚`
    - 19:00–24:00：`夜间`

## 文案生成与降级
- 大模型生成
  - 投喂内容（建议结构化）：
    - 名称：`{name}`（昵称或“同学”）
    - 校区：`{campus}`
    - 天气：`{summary}`（可空）与 `{temp}`（可空）
    - 时段：`{timeSlot}`
  - 指令约束（要点）：
    - 使用简体中文；输出单句；≤30 字；不含 emoji；标点≤2。
    - 风格自然友好；避免命令口吻；允许轻微提醒（如下雨带伞）。
    - 若天气缺失，避免硬拼接；语句需通顺。
- 降级模板
  - 完整信息可用：`{name}你好，现在是{timeSlot}，{campus}{summary}{tempHint}，祝你顺利。`
  - 天气缺失（`summary` 空）：`{name}你好，现在是{timeSlot}，祝你顺利。`
  - `tempHint`：若存在温度数据，用 `，{temp}`；否则为空。
  - 生成后同样执行长度与字符校验，必要时截断至 30 字。

## 信息架构与展示
- 展示位置：首页页面顶部区域，置于导航/功能区之上。
- 形态：单行主文案；最多两行展示，超出以省略号截断（但生成目标≤30字）。
- 状态：
  - 加载中：骨架文本或淡入占位（约 1–2s 以内）。
  - 已生成：显示招呼语；右侧提供“刷新”小图标按钮（可选）。
  - 失败：显示降级模板或固定兜底文案“欢迎回来，祝你学习顺利”。

## 交互与状态流
- 触发
  - 首次进入首页：读取缓存 → 若缓存失效或不存在 → 拉天气 → 调大模型 → 展示。
  - App 回到前台：距离上次生成 ≥30 分钟时重复上述流程。
  - 手动刷新：按钮点击立即触发重新拉取与生成（忽略缓存）。
- 成功
  - 渲染新文案 → 写入本地缓存（含 signals 与 content） → 更新 `createdAt`。
- 失败
  - 天气失败：使用空天气继续大模型生成；若仍失败，落入模板。
  - 大模型失败：直接使用降级模板。
  - 所有失败均保证最终有可展示文案。

## 非功能性（简版）
- 性能：本地信号处理 < 50ms；天气请求 + 生成整体首屏 ≤ 1.2s（弱网可至 2s）。
- 离线：无网络时直接使用降级模板；不阻塞首页其他内容渲染。
- 可用性：从进入到看到文案的核心路径不超过 1 次点击（可选的刷新）。
- 隐私与合规：不上传真实身份标识到日志；调用大模型仅传昵称与城市级信息。
- 稳定性：接口失败有明确降级；缓存失效可自愈；异常不崩溃。

## 验收标准（关键用例）
- 首次进入：在首页顶部出现一条中文招呼语，≤30 字，语气自然。
- 天气可用：文案包含校区与天气要素，如“小雨/多云”等，语句通顺。
- 天气缺失：文案仍通顺，不出现“undefined/空”之类字符。
- 时段准确：在 12:30 显示“中午”，在 15:00 显示“午后”，在 20:00 显示“夜间”。
- 名称策略：用户设置了昵称则使用昵称；未设置显示“同学”。
- 切换校区：用户在设置中变更校区后返回首页，文案随之更新（可点击手动刷新）。
- 长度限制：任何返回文案最终展示 ≤30 字；超出将安全截断。
- 失败降级：断网或大模型失败时展示模板文案，不影响首页使用。
- 刷新行为：点击刷新按钮后 2s 内看到新文案或降级文案（弱网除外）。

## 对接说明（工程侧）
- 组件入口：`src\modules\Home\components\Greeting\index.tsx`
- 依赖与接口（示意）
  - `weatherService.getSummary(campus: string): Promise<{ summary: string; temp?: string }>`
  - `timeSlot.getLabel(date: Date): '凌晨'|'清晨'|'上午'|'中午'|'午后'|'傍晚'|'夜间'`
  - `llm.generateGreeting(payload: { name: string; campus: string; summary?: string; temp?: string; timeSlot: string }): Promise<string>`
- 事件（内部行为）
  - `load()`：组件挂载时拉取与生成。
  - `refresh()`：用户点击刷新或应用回前台触发。
  - `onCampusChanged()`：监听用户校区变更，触发重新生成。
- 缓存策略
  - Key 由 `campus + timeSlot` 组成；有效期 30 分钟；手动刷新忽略缓存。
- 安全校验
  - 对返回文案执行：字符集校验（中文/常见标点）、长度截断、敏感词过滤、emoji 过滤。
- UI 建议
  - 单行主文案，字体 16–18sp；预留最多两行高度；右侧小号“刷新”图标按钮。
  - 骨架或淡入动画以提升感知速度。

## 提示词建议（供实现参考）
- 指令要点（合并成一段传入大模型）：
  - 请用简体中文输出一条 ≤30 字的问候语；不要使用 emoji；标点不超过 2 个；语气自然友好。
  - 输入信息：姓名（昵称或“同学”）：{name}；校区：{campus}；时段：{timeSlot}；天气：{summary}{temp?}。
  - 若天气缺失，不要硬拼接；语句需通顺；可有礼貌提醒，如遇雨提示带伞。
