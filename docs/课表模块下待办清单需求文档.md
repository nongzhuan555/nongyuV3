# 课程待办清单（简化版 PRD）

## 文档信息
- 文档版本：v1.0-简化
- 对接组件：`src\modules\Course\components\TodoList\index.tsx`
- 适用平台：App（安卓和 iOS）

## 产品原则
- 简洁、直接、低学习成本。
- 每门课程自有独立清单；仅“待办/已办”两态；最少字段与最少交互。

## 范围与边界
- In Scope
  - 每门课程拥有一份独立的待办清单。
  - 待办清单分为两个区域：待办区、已办区。
  - 每条待办仅包含：创建时间、完成时间、当前状态、文本内容。
  - 基本交互：新建、完成/撤销、删除、查看详情（弹窗）。
  - 删除触发方式：长按出现操作按钮；或左滑出现删除按钮；删除前必须确认。
- Out of Scope（本期不做）
  - 优先级、截止时间、提醒、标签、批量操作、筛选/排序、分组、日历视图、附件等。
  - 教师/助教视角与协作能力。

## 核心概念与数据模型
- Todo（待办）
  - 字段
    - `id`: string
    - `courseId`: string（归属课程）
    - `content`: string（待办的文字内容）
    - `status`: `'todo' | 'done'`（仅两种）
    - `createdAt`: ISO string（创建时间）
    - `completedAt`: ISO string | null（完成时间，未完成为 null）
  - 约束
    - 新建后立即处于 `todo`，写入 `createdAt`。
    - 标记完成时置为 `done`，写入 `completedAt`。
    - 撤销完成置回 `todo`，`completedAt` 置为 null。
    - 任何状态均可删除，但必须二次确认。

## 信息架构与展示
- 列表结构
  - 页面内按课程展示对应清单，分为“待办区”和“已办区”两块。
  - 每条待办使用固定高度卡片纵向排列；卡片显示内容文本和简要时间信息。
  - 文本过长时以省略号截断（最多两行）；点击卡片打开弹窗查看完整内容。
- 区域排序
  - 待办区：按 `createdAt` 倒序（最新在上）。
  - 已办区：按 `completedAt` 倒序（最近完成在上）。

## 交互与状态流
- 新建
  - 入口：清单页右下角按钮或顶部入口（由实现决定，其一即可）。
  - 表单：仅一个多行文本输入框（content）。
  - 提交：创建 `todo`，写入 `createdAt`，插入待办区顶部。
- 完成与撤销
  - 单条操作：卡片上的勾选/按钮切换状态。
  - 完成：移入已办区并记录 `completedAt`。
  - 撤销：移回待办区并清空 `completedAt`。
- 删除
  - 触发方式：
    - 长按卡片：出现操作按钮，点击“删除”。
    - 左滑卡片：出现“删除”行动按钮。
  - 确认：弹出确认 Modal（文案：“确认删除该待办？删除后不可恢复。”），确认后删除。
- 查看详情
  - 点击卡片：弹出只读弹窗展示待办完整内容与时间信息。

## 非功能性（简版）
- 性能：列表滚动流畅；单次操作反馈 < 150ms。
- 离线：当前阶段使用前端本地状态（或持久化存储可选），不依赖后端。
- 可用性：主要路径不超过 2 步（创建/完成/删除/查看）。

## 验收标准（关键用例）
- 创建：输入文本后创建一条待办，出现在“待办区”顶部，`createdAt` 有值、`status=todo`。
- 完成：点击完成后该项进入“已办区”，`status=done` 且 `completedAt` 有值。
- 撤销：在“已办区”对条目撤销后回到“待办区”，`status=todo` 且 `completedAt=null`。
- 删除（长按）：长按卡片出现“删除”，点击后弹出确认，确认后条目消失。
- 删除（左滑）：左滑出现“删除”，点击后弹出确认，确认后条目消失。
- 查看详情：点击任意卡片，弹出详情弹窗展示完整内容与时间信息。
- 文本溢出：列表卡片超出两行时以省略号展示。

## 对接说明（工程侧）
- 组件入口：`src\modules\Course\components\TodoList\index.tsx`。
- 状态：前端本地管理即可（可选使用持久化，如 AsyncStorage），无需后端。
- 事件（内部行为）：`create(content)`、`toggleDone(id)`、`delete(id)`、`openDetail(id)`。
- UI 建议：固定卡片高度；卡片含多行文本+时间；长按与左滑行为取其一即可优先实施，另一种作为增强补充。
