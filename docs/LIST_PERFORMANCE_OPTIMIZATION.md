# 二课活动列表滚动性能优化方案

本文档描述了在二课活动列表（`ActivityList/index.tsx`）中实施的滚动性能优化策略。当列表项较多时，这些优化措施能显著提升滚动流畅度和内存使用效率。

## 1. 组件级优化

### 1.1 列表项组件 Memoization
**实施方式**: 使用 `React.memo` 包装 `ActivityItem` 组件。
**原理**: 只有当 `item` 数据或 `onPress` 函数发生变化时，列表项才会重新渲染。这避免了父组件（列表页）状态更新（如 loading 状态变化）导致所有列表项不必要的重渲染。

```typescript
const ActivityItem = React.memo(({ item, onPress }: { item: Activity; onPress: (id: number) => void }) => {
  // ...
});
```

### 1.2 渲染函数 Memoization
**实施方式**: 使用 `useCallback` 包装 `renderItem` 函数。
**原理**: 确保传递给 `FlatList` 的 `renderItem` 引用在多次渲染间保持稳定，配合 `React.memo` 发挥最大效用。

```typescript
const renderItem = useCallback(({ item }: { item: Activity }) => (
  <ActivityItem item={item} onPress={...} />
), [navigation]);
```

## 2. FlatList 配置优化

### 2.1 初始渲染数量 (`initialNumToRender`)
**配置值**: `10`
**说明**: 指定首屏渲染的列表项数量。设置得当可以减少首屏加载时间，避免一次性渲染过多不可见的内容。

### 2.2 批量渲染数量 (`maxToRenderPerBatch`)
**配置值**: `10`
**说明**: 决定每批次增量渲染的元素数量。由于我们的列表项结构相对复杂（包含图片、多个Chip、文本），限制每批次渲染数量可以避免 JS 线程阻塞，保持滚动流畅。

### 2.3 渲染窗口大小 (`windowSize`)
**配置值**: `5` (默认通常是 21)
**说明**: 这是一个无单位的整数，表示可视区域高度的倍数。设置为 5 意味着仅保留可视区域上下各 2 个屏幕高度的内容在内存中。
**收益**: 显著降低内存占用，特别是当列表包含大量图片时。虽然可能导致快速滚动时出现短暂白屏，但在内存受限的移动设备上是必要的权衡。

### 2.4 裁剪子视图 (`removeClippedSubviews`)
**配置值**: `true`
**说明**: 启用后，超出可视区域的视图会被卸载（在 Android 上效果显著）。这能大幅减少原生视图层级的内存占用。

## 3. 图片优化 (已存在)

### 3.1 图片尺寸控制
**实施方式**: `resizeMode="cover"`
**说明**: 确保图片在容器内正确缩放，避免渲染过大的纹理。

## 4. 后续可优化项 (待定)

如果列表性能仍有瓶颈，可考虑以下进一步优化：

1.  **getItemLayout**: 如果列表项高度固定，实现此方法可以跳过动态高度计算，大幅提升滚动定位性能。目前由于标题行数不固定（1-2行），暂未实施。
2.  **图片懒加载**: 使用第三方库或自定义组件，仅当图片进入可视区域时才开始网络请求。
