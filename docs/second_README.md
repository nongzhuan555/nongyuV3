# 二课系统架构设计
### 大体架构
点击进入二课系统前，需要通过检查本地存储中是否包含二课的token字段，如果没有包含，则需要弹出一个二课登录页用来登录二课系统，登录成功后则会保存返回的token信息在本地用于后续其他请求的鉴权同时也是用户已经登录过的标志，另外，如果调用二课的接口时发现登录过期相关的提示，那么app自动弹出登录页重新登录获取token，然后继续之前的请求。
### 功能概览
- 二课活动列表（以列表卡片的形式展示二课活动，包括活动的名称、时间、地点等信息，具体的参考下方接口文档，活动列表页可模糊搜索活动，也可以按照一定规则进行活动的筛选（按照部落进行筛选或按照活动分类进行筛选），具体的可筛选条件同样参考下方接口中给出的接口）
- 二课活动详情页（点击活动列表中的某一个活动，进入该活动的详情页，展示该活动的详细信息，包括活动的时间、地点、活动的参与人数等信息，具体也去取决于下方接口文档中给出的接口）
- 用户个人的二课信息，包括二课分详情、二课分分布（如德育部分多少分等）、二课排行榜信息展示
